Document and signature scanner

This real-world project helps in something we deal with on a daily basis - to export documents and signature to either re-use them or have a soft copy of a printed document. This has been an interesting Computer Vision project to explore principles of edge detection, finding large contour, and applying perspective transform to flatten the document.

This project looks at 1,800 photos to train in detecting text and exporting these to an editable format. Source of the dataset is from the project https://universe.roboflow.com/doc-scan/doc-scanner. This project looks considers this raw data, processes it, with examples of tests saved. Further experiments are performed to detect edges and contours. Further, the source code contains `scanner.py` to detect edges, crop and transform, `utils.py` to have helper functions, `pipeline.py` to describe the end-to-end process of loading, scanning, and extracting editable text from the document images.

Steps for development

  1. Data preparation:
    • Used the dataset from the project https://universe.roboflow.com/doc-scan/doc-scanner, which contains 1,800 document images. These are saved in the `raw` folder, indicating raw data.
    • These are then processed to remove noise, so it is easier to detect edges, contours, and is potentially ready for transformation.
  2. Edge detection, contours, and trasnformation:
    • This is the core functionality of this project, where the principles of Computer Vision are used to detect edges, contour, and transform images, so text could be extracted with utmost accuracy.
  3. App development
    • It is anticipated to create an app to allow users to try uploading a document and get back an editable copy of their document.
  4. CI/CD pipeline using Jenkins and containerisation using Docker
    • Jenkins is super helpful for automatic testing, linting and style checks.
    • Potentially, the app could be packages as a Docker image.
